Service Implementation guide

Step 1: Prepare your servers

  • Create two new Google Cloud instances
    • First Instance: Bitnami’s LAMP stack from the GCP Marketplace.
      • Hosts informational website and OAuth SSO Server Plugin
    • Second Instance: Bitnami’s WordPress LAMP stack from the GCP Marketplace.
      • Hosts Tiny File Manager and WordPress SSO “Client” Plugin
    • For the sake of this tutorial, we will call the instance that hosts Tiny File Manager TFM-Instance, and the one that hosts the website and the WordPress SSO server will be referenced as WEB-Instance.  
  • Modify the GCP firewall rules on your instances to allow HTTP and HTTPS traffic.
  • Ensure your systems are up to date:
    • sudo apt update
    • sudo apt upgrade

Step 2: Download Required Software Packages

  1. Download WordPress on both instances:
    1. cd /tmp
    2. wget https://downloads.bitnami.com/files/stacks/wordpress/X.Y.Z/bitnami-wordpress-X.Y.Z-module-linux-x64-installer.run
    3. chmod +x ./bitnami-wordpress-X.Y.Z-module-linux-x64-installer.run
    4. sudo ./bitnami-wordpress-X.Y.Z-module-linux-x64-installer.run
  2. [TFM-Instance] Download our implementation of Tiny File Manager:
    1. git clone https://github.com/saltrupiao/SaaSFileManager.git

Step 3: Implement Tiny File Manager (TFM-Instance)

  1. SSH into the TFM-Instance virtual machine.
  2. Navigate to the directory where you cloned the Tiny File Manager application files.
  3. Move the application folder into the document root of WordPress:
    1. sudo mv ./SaaSFileManager/control-panel/ /opt/bitnami/apps/wordpress/htdocs/.
  4. Modify permissions of the application directory:
    1. cd /opt/bitnami/apps/wordpress/htdocs
    2. sudo chmod 755 control-panel
  5. Create a directory for Tiny File Manager to use for file management:
    1. sudo mkdir files
    2. sudo chmod 777 files

Step 4: Configure WordPress on WEB-Instance

  1. Login to the WordPress Dashboard at: http://[SERVER_IP]/wordpress/wp-admin using the default credentials.
  2. Change your administrator password for security purposes.
  3. Install the following Plugins:
    1. WP OAuth Server (SSO Server)
    2. Elementor (WP Page Editor)
  4. Create a default index page with a login button that links to the following address: http://[TFM-Instance IP]/?auth=sso

Step 5: Configure WordPress on TFM-Instance

  1. Login to the WordPress Dashboard at: http://[SERVER_IP]/wp-admin using the default credentials.
  2. Change your administrator password for security purposes.
  3. Install the following Plugins:
    1. Simple Single Sign On (SSO Client)
    2. Elementor (WP Page Editor)
    3. My Private Site (Part of Securing Tiny File Manager)
  4. Create a landing page – this is where the user will be taken after successfully authenticating with SSO.
    1. Somewhere on this landing page, include a prominent button that the user clicks on to launch the Application. The button should link to this address: http://[TFM-Instance IP]/control-panel/index.php
  5. Configure My Private Site Plugin:
    1. Mouse over Settings on the left sidebar of the WordPress Dashboard and click on Installed Plugins.
    2. Under My Private Site in the list of plugins, click Settings.
    3. Check the box next to Private Site to enable the plugin, click Save Changes when checked.
    4. In the Specific URL dialog box, enter: http://[TFM-Instance IP]/control-panel/index.php
    5. Under Visible Exclusions, check the box next to Site Home Always Visible?
    6. Click Save Changes.

Step 6: Configure SSO On WEB-Instance

  1. Login to the WordPress Dashboard at: http://[SERVER_IP]/wp-admin using admin credentials.
  2. Mouse over OAuth Server in the left sidebar and select Clients.
  3. Click Add New Client near the top of the page.
  4. Enter the following information inside the Client Information form:
    1. Client Name: TinyFileManager
    2. Redirect URI: http://[TFM-Instance IP]/?auth=sso
  5. Check Authorization Code on the left of the page.
  6. Click Create Client.
  7. You will then get a generated Client ID and Client Secret. Note these down, as you will need them in the next step.

Step 7: Configure SSO on TFM-Instance

  1. Login to the WordPress Dashboard at: http://[SERVER_IP]/wp-admin using admin credentials.
  2. Mouse over Settings in the left sidebar and click on Single Sign On
  3. Click on the  Step 2: Configuration dropdown.
  4. Enter the Client ID and Client Secret you received in the previous step.
  5. For OAuth Server URL, type: http://[WEB-INSTANCE IP]/wordpress/
  6. Click Save Changes.

Step 8: User Configuration

Users of this service may create an account using the WordPress Account Registration service.

If this was going into production, we would ideally implement a step with a billing API, where they have to enter credit card information to pay for this Software-as-a-Service. 

All administrator user account configuration occurs in the Dashboard of the WordPress Dashboard of WEB-INSTANCE.

All user accounts can be created and managed by navigating to the Users page in the WordPress Dashboard.
